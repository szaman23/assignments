Want to learn how to create accessible websites? Then this beginner's tutorial is for you!

Why make accessible websites?

A developer needs to ensure that users with disabilities (such as color blindness) can easily view and interact with websites.

Some of the disabilities which need to be taken into account when designing a website are:

  1. Poor vision-such as color blindness
  2. Seizures- usually caused by flashing lights
  3. Hearing problems
  4. Mobility issues-difficulties with fine motor skills e.g. using a keyboard

Making websites accessible not only makes it easy for disabled users, it also increases the overall quality of a website. This tutorial will give you some tips on how to make an accessible website.

Tip 1: Text alternatives for images

To make a website accessible, you should use text alternatives for images. If an image cannot be displayed on a browser or if a user is disabled, then a screen reader will read the alt text instead. This will provide users with some idea of what the image is about. So the alt text must describe the image used in the website. The first image doesn't exist, but since we used an "alt" tag, we can see the description of the image.

Tip 2: Using structural tags

HTML has 6 tags, ranging from h1-h6 and it is important to use them correctly. Screen readers use the heading tags to read through the content, so the tags should be organized in a specific order. Check out these examples to learn more.

Use h1 for headings only. h1 should only describe the title of a page.

h2 size is less than h1. This means that the importance of each tag decreases as we go down the list.

This is h3. DO NOT skip between tags (such as using h1 and then h3) the screen reader will think that something is missing in between.

Tip 3: Defining high contrast

Color blind (or color deficient) people have difficulty seeing some colors. For example, some people have red-green, or blue-yellow color blindness. This means that they will not be able to see red text on a green background, and vice versa. For this reason, it is important to use colors which color blind people can see. This can be done by using a high contrast ratio.

The text can hardly be seen! Bad contrast!

The text can be seen clearly. Good contrast.

Tip 4: Organizing Menu links

Users who are blind or cannot use keyboard rely on screen-readers to interact with a website. So menu links should be described clearly. Words such as "Click here!" or "Find out more!" which do not describe the menu should not be used.Below is an example of an organized menu.

  • Home
  • About
  • Contact

Tip 5: All about tables

By simply including a caption in a table, you can make your website more accessible to disabled/blind users. When the screen reader reads the caption, the user will have an idea of what the table is about. You should also add headers to your table to represent columns. Screen readers read the header in a different way, so blind users will be able to know which data represents rows and columns.

Some other tips

There are many other things to keep in mind while designing an accessible website. Some of these include:

  1. Using subtitles/closed captioning for videos
  2. Not having lights flash for longer than a specific threshold
  3. Not using small fonts

And many more! After all, there are 3 levels of conformance for accessibility- A (lowest), AA, and AAA (highest). For more information, check out the guidelines at WAI.
